# 管理后台使用说明

## 功能介绍

管理后台提供了一个可视化的界面，用于上传和管理音乐播放器的歌曲资源。

## 架构说明

管理后台采用前后端分离架构：
- **前端**：使用 React 技术栈，集成在 `music` 项目中
- **后端**：提供 RESTful API 接口，使用 Express + MySQL

## 访问地址

### 前端访问

启动前端服务后，在浏览器中访问：

```
http://localhost:5173/admin
```

### 后端 API

后端服务运行在 `http://localhost:3000`

## 主要功能

### 1. 上传新歌曲

- **必填字段**：歌曲标题、艺术家、音频文件
- **可选字段**：专辑、时长、封面图片
- **支持的音频格式**：mp3, wav, ogg, m4a
- **支持的图片格式**：jpg, jpeg, png, gif, webp
- **文件大小限制**：最大 50MB

### 2. 查看歌曲列表

- 显示所有已上传的歌曲
- 包含封面预览、标题、艺术家、专辑、时长等信息
- 按创建时间倒序排列

### 3. 编辑歌曲信息

- 点击"编辑"按钮修改歌曲信息
- 可修改标题、艺术家、专辑、时长
- 不支持修改音频文件和封面图片

### 4. 删除歌曲

- 点击"删除"按钮删除歌曲
- 删除时会同时删除数据库记录和上传的文件
- 删除歌曲会自动从所有歌单中移除

## 技术实现

### 前端技术栈

- React 18
- React Router
- CSS Modules
- Fetch API

### 后端技术栈

- Express.js
- MySQL2
- Multer (文件上传)

### 文件上传

使用 `multer` 中间件处理文件上传：
- 音频文件存储在 `backend/uploads/songs/` 目录
- 封面图片存储在 `backend/uploads/covers/` 目录
- 文件名使用时间戳 + 随机字符串，避免重复

### API 接口

| 方法 | 路径 | 说明 |
|-----|------|------|
| GET | `/api/admin/songs` | 获取所有歌曲列表 |
| POST | `/api/admin/songs` | 上传新歌曲 |
| PUT | `/api/admin/songs/:id` | 更新歌曲信息 |
| DELETE | `/api/admin/songs/:id` | 删除歌曲 |

### 静态文件访问

- 上传的文件通过 `/uploads` 路径访问
- 例如：`http://localhost:3000/uploads/songs/song_123456.mp3`

## 使用示例

### 上传歌曲

1. 点击侧边栏的"管理后台"进入管理页面
2. 点击"上传歌曲"按钮
3. 填写歌曲信息（标题、艺术家为必填）
4. 选择音频文件
5. （可选）选择封面图片
6. 点击"上传"按钮
7. 等待上传完成，歌曲会自动添加到下方列表

### 编辑歌曲

1. 在歌曲列表中找到要编辑的歌曲
2. 点击"编辑"按钮
3. 在弹出的模态框中修改信息
4. 点击"保存"按钮

### 删除歌曲

1. 在歌曲列表中找到要删除的歌曲
2. 点击"删除"按钮
3. 在确认对话框中点击"确定"

## 项目结构

```
│   └── src/
│       ├── controllers/

│       └── routes/
│           └── adminRoutes.js         # 管理后台路由
│
└── music/                         # 前端项目
    └── src/
        └── components/
            └── views/
                └── AdminView.jsx    # 管理后台 React 组件
                └── AdminView.css    # 管理后台样式
```

## 注意事项

1. 删除歌曲后，所有包含该歌曲的歌单会自动移除该歌曲
2. 不支持修改已上传的音频文件和封面图片，如需修改请删除后重新上传
3. 文件上传大小限制为 50MB，如需修改请编辑 `backend/src/routes/adminRoutes.js` 中的配置
4. 上传的文件存储在服务器本地，请确保有足够的磁盘空间

## 安全建议

1. 在生产环境中，建议为管理后台添加身份验证功能
2. 限制文件上传的大小和类型
3. 定期清理无用的上传文件
4. 考虑将文件上传到云存储服务（如阿里云 OSS、腾讯云 COS）
5. 使用 HTTPS 传输，确保数据安全
